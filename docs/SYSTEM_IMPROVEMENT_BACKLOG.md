# Backlog การพัฒนาระบบ THEELIN (Office / ไซต์ / การเงิน)

**หน้าในระบบ (admin):** `pages/internal/system-improvement-hub.php` — ลิงก์จากเมนูโปรไฟล์ → «แผนพัฒนาและงานค้าง»

เอกสารนี้สรุปจากการวิเคราะห์ workflow และโครงสร้างโมดูล — ใช้คุยใน meeting และติดตามงานได้

**วิธีใช้:** ติ๊ก `[ ]` → `[x]` เมื่อทำเสร็จ | เติมชื่อผู้รับผิดชอบในวงเล็บ

## P1 — พัฒนาระบบให้ทำงานร่วมกันชัดขึ้น

| # | รายการ | หมายเหตุ | ผู้รับผิดชอบ |
|---|--------|----------|--------------|
| 1 | [ ] **ศูนย์กลาง “งานค้าง”** หรือแดชบอร์ดสรุปสถานะ | รวมรายการรออนุมัติ / รอเอกสาร จากหลายโมดูล (อย่างน้อยลิงก์รวมจากหน้าแรก) | ( ) |
| 2 | [ ] **แผนผังเอกสาร 1 หน้า** (Quotation → Invoice → ใบกำกับ / PR → PO → Bill → Stock?) | แปะใน Confluence/Notion หรือ PDF ก็ได้ | ( ) |

| 4 | [ ] **Traceability** — อ้างอิงย้อนกลับระหว่างเอกสาร | จากใบหนึ่งไปหาเอกสารต้นทาง (ตามความจำเป็นต่อธุรกิจ) | ( ) |

## โมดูล — จุดที่ควรยืนยันกับทีม (ไม่ใช่แค่แก้โค้ด)

| โมดูล | คำถามที่ต้องตอบร่วมกัน |
|--------|-------------------------|
| จัดซื้อ + สต็อก | PO / บิลซื้อ **ตัดสต็อก** อัตโนมัติหรือมือ? |
| การเงิน | Advance → ออกใบเสร็จ → ลง petty / รายงาน อย่างไร? |
| ค่าแรง / ไซต์ | ต้องการ **ผูกต้นทุนต่อไซต์** ในระบบหรือไม่? |
| ลา / payroll | ขั้นอนุมัติลา **สอดคล้อง** กับการจ่ายเงินเดือนหรือไม่? |

## UX / ความสม่ำเสมอ (ทำได้ทีละน้อย)

- [ ] ใช้ pattern เดียวกันในสายงานเดียวกัน (เช่น list + popup vs หน้าเต็ม)
- [ ] หลังบันทึกสำเร็จ — ข้อความยืนยัน + เส้นทางกลับชัด (หลายจุดทำแล้ว — ขยายเป็นมาตรฐาน)
- [ ] ตรวจชื่อเมนูภาษาไทย–อังกฤษให้ตรงกับที่ทีมเรียกจริง
